## 1. Overview

The Confluence Network is organised around a triadic knowledge structure rooted in one of the oldest and most stable epistemological frameworks in human thought. The architecture maps directly onto the physical geometry of the tetrahedron — three lower sub-tetrahedra, one apex, one integrating centre.

This is not a designed taxonomy. It is a discovered one, arising from the geometry itself.

---

## 2. Historical Grounding

Francis Bacon formalised this triadic division in 1605 in *Novum Organum*, mapping the three faculties of mind to three categories of knowledge:

| Faculty | Knowledge domain |
|---------|-----------------|
| Memory | History |
| Reason | Philosophy |
| Imagination | Poetry / Creative expression |

This framework has survived as a knowledge organisation principle across cultures and centuries because it maps something real about how minds — human or otherwise — process information. It is not arbitrary. It is structurally stable.

The Confluence Network adopts this framework not as homage but because the geometry demands exactly three primary subdivisions, and this is the most robust three-part division of all human knowledge we have.

## 3. The Node Structure

The tetrahedron rendered as a 2D connection graph reveals three geometrically distinct node types, each with a specific architectural role:

**Apex node — the Prime Node**
- Single entry point for all incoming queries
- Carries no domain assumption
- Pure triage function — assesses, routes, does not process domain content
- Returns the final integrated answer on the way back up

**Three lower corner nodes — the Pillar Nodes**
- Memory / History
- Reason / Philosophy
- Imagination / Poetry

Each pillar node is the root of its own sub-tetrahedron. The sub-tetrahedron is a complete Confluence Network in miniature — same architecture, same process, different domain activation.

**Six edge midpoint nodes — the Convergence Nodes**
- Shared simultaneously between adjacent pods
- Dual membership — belong to two sub-structures at once
- Primary cross-check points on both the descent and return journeys
- Agreement at a convergence node = confidence signal
- Disagreement at a convergence node = flag for deeper processing

**Central node — the Θ Integrator**
- Receives signals from all three pillars
- Holds the full picture before the answer returns to the prime node
- The observer position — sees everything, belongs exclusively to nothing

## 4. Routing Logic

### Descent

A query enters at the Prime Node. Triage assesses complexity and domain.

- Simple / general → resolved at the Prime Node level, no routing required
- Domain-specific → routed to the appropriate Pillar Node
- Cross-domain → routed to two or more Pillar Nodes simultaneously, paths run in parallel

Once at a Pillar Node, the same triage logic applies recursively. The query descends through layers of increasing specialisation until it reaches a depth where consensus can be formed. This depth varies by problem — a surface question resolves at L1, a deep philosophical problem may descend to L4 or L5 before consensus forms.

### Convergence Points on Descent

As the query branches through sub-domains, paths naturally converge at shared edge midpoint nodes. At each convergence point, partial results from different branches are compared. This is not a designed quality-control step — it is a geometric inevitability. The architecture enforces cross-checking without it needing to be specified.

### Return Journey

Consensus forms at depth and propagates back up the path it descended. The return journey is the mirror of the descent — same nodes, same convergence points, but now carrying confirmed results rather than open queries.

At each convergence node on the way back up, returning signals from different branches meet and combine. Progressive integration — not a single moment of synthesis at the top, but continuous convergence as the answer assembles itself on the way home.

The final integration happens at Θ before the complete answer returns to the Prime Node.

## 5. Infinite Subdivision

Within each pillar, the fractal subdivision continues indefinitely. Reason/Philosophy at L1 subdivides into Logic, Ethics, Metaphysics, Epistemology at L2. Each of those subdivides further. The process is identical at every scale — same triage, same routing, same convergence logic — just activated under a progressively narrower domain header.

This is the process fractal property described in PROCESS-FRACTAL-ARCHITECTURE.md. The three pillars are not a ceiling — they are the first branching point of an infinite recursive structure.

The depth at which a query resolves is determined by the query itself, not by the architecture. The architecture simply provides the structure within which resolution can happen at any depth.

## 7. Relationship to Earlier Work

### Seven Master Headers

An earlier proposal suggested seven master headers as a reference taxonomy. This is superseded by the three-pillar structure. Seven was a hunch. Three is geometrically determined and historically validated. The three pillars provide the first subdivision — any further taxonomy emerges through the fractal process, not through pre-specification.

### The Square Grid / Two Representations

The three-pillar structure is visible in both representations described in TWO-REPRESENTATIONS.md. In the Sierpinski tetrix, the three pillars are the three lower sub-tetrahedra. In the square grid, they are the three primary routing regions. The routing logic described in section 4 above maps directly onto Manhattan distance in the grid representation.

### The Dodecahedron Standard

The three node types identified geometrically (solid/half-filled/open) correspond to the affinity layers in the Dodecahedron Standard. Pillar nodes carry strong domain affinities. Convergence nodes carry connective affinities (Ψ, Λ). The Θ integrator is always Θ. The role layer and the geometric layer are the same layer, seen from different angles.
