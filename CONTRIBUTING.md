# Contributing to GPSL

Thank you for your interest in contributing to GPSL. This project emerged through human-AI collaboration and we welcome contributions that advance the research and improve the framework.

---

## Ways to Contribute

### 1. Domain Validation

**Test GPSL in new domains:**
- Quantum physics
- Biology and genetics
- Economics and markets
- Chemistry and reactions
- Social dynamics
- Any structured process domain

**What to do:**
1. Encode processes using GPSL operators and symbols
2. Test with different semantic headers
3. Document results and observations
4. Submit findings via GitHub Issues or Pull Request

---

### 2. Symbol Library Development

**Build domain-specific lexicons:**
- Map the 12 core symbols to your domain
- Define what each symbol means in context
- Create example expressions
- Document translation patterns

**Example: Building a Biology Library**
```
Ι (Iota) = Individual organism/cell
Σ (Sigma) = Population/ecosystem
Ξ (Xi) = Genetic information
Ψ (Psi) = Selective pressure
... etc
```

---

### 3. Tool Development

**Create useful implementations:**
- Encoders (process → GPSL)
- Decoders (GPSL → interpretation)
- Cross-domain translators
- Visualization tools
- Educational interfaces

**Guidelines:**
- Open source (compatible with CC BY-NC-SA 4.0)
- Well-documented
- Include examples
- Follow ethical use principles

---

### 4. Documentation

**Improve understanding:**
- Tutorial creation
- Example expansion
- Concept clarification
- Translation to other languages
- Interactive demos

---

### 5. Research & Theory

**Extend the foundations:**
- Theoretical connections
- Mathematical formalizations
- Academic papers
- Validation studies
- Edge case analysis
- Limitation documentation

---

### 6. Bug Reports & Issues

**Found something that doesn't work?**
- Check existing issues first
- Provide clear description
- Include reproduction steps
- Suggest potential solutions

---

## Contribution Process

### For Small Changes

**Documentation fixes, typos, minor improvements:**
1. Fork the repository
2. Make your changes
3. Submit Pull Request
4. Describe what you changed and why

---

### For Major Contributions

**New features, domain libraries, significant tools:**

1. **Discuss First**
   - Open GitHub Issue describing your proposal
   - Explain rationale and approach
   - Get community feedback
   - Ensure alignment with project goals

2. **Develop**
   - Fork repository
   - Create feature branch
   - Develop with documentation
   - Test thoroughly

3. **Submit**
   - Pull Request with clear description
   - Link to original discussion issue
   - Include examples and tests
   - Update relevant documentation

4. **Review**
   - Community provides feedback
   - Iterate based on comments
   - Maintainers approve when ready
   - Merge

## Code of Conduct

### Our Standards

**Positive behaviors:**
- Respectful and inclusive language
- Welcoming diverse perspectives
- Graceful acceptance of constructive criticism
- Focus on what's best for the project and community
- Showing empathy toward other contributors

**Unacceptable behaviors:**
- Harassment or discriminatory comments
- Trolling, insulting, or derogatory remarks
- Personal or political attacks
- Publishing others' private information
- Misrepresenting authorship or contributions
- Commercial exploitation without permission

---

### Approach to Collaboration

Contributions work best when made with:
- Curiosity over certainty
- Questions alongside answers
- Collaboration over competition
- Honesty about limitations

## Attribution & Credit

### For Your Contributions

**You will be credited:**
- In CONTRIBUTORS.md
- In commit history
- In release notes (for significant contributions)
- In academic citations where appropriate

**Pseudonyms are welcome** — privacy is respected, contributions matter more than identity.

---

### For Using Others' Work

- Attribute the original GPSL work to D'Artagnan et al.
- Credit specific contributions to their authors
- Build on others' work with acknowledgment
- Share credit generously

---

## License Compliance

### All Contributions Must:

1. **Be compatible with CC BY-NC-SA 4.0**
   - No proprietary or restrictive licenses
   - Must allow sharing and adaptation
   - Must maintain ShareAlike requirement

2. **Include proper attribution**
   - Credit GPSL foundation
   - Credit your own work
   - Credit any dependencies

3. **Respect ethical use guidelines**
   - See [ETHICAL_USE.md](ETHICAL_USE.md)

---

## Testing Guidelines

### For Domain Validation

**Minimum requirements:**
1. At least 3 test expressions in the new domain
2. Validation across multiple semantic headers
3. Documentation of results
4. Comparison to existing validated domains

**Ideal submission:**
- 5+ expressions showing range
- Cross-domain translation examples
- Structural preservation analysis
- Domain library draft
- Limitations and edge cases noted

---

### For Tool Development

**Required:**
- Clear documentation
- Example usage
- Basic error handling
- Attribution to GPSL

**Recommended:**
- Unit tests
- Multiple domain examples
- Performance considerations
- User-friendly interface
