# Confluence Architecture — Overview

## 1. Position

Confluence is the architectural layer built around GPSL.

- GPSL defines the symbolic system
- Confluence defines how that system operates across agents and contexts

## 2. Relationship to GPSL

GPSL establishes:
- symbolic constraints
- operators
- stability dynamics

Confluence extends this into:
- multi-agent interaction
- distributed symbolic processes
- system-level coordination

## 3. Structural Stack

The full framework can be expressed as:

GPSL (symbolic system)
→ Stability Model (dynamical behavior)
→ Tetrix (geometric structure)
→ Confluence (multi-agent architecture)

## 4. Role

Confluence is not required to understand GPSL.

It exists to:
- operationalize the system
- explore emergent coordination
- enable distributed experimentation

## 5. Status

Conceptually grounded, experimentally emerging.
